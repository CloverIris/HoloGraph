# HoloGraph 产品需求文档 (PRD)

## 产品概述

### 产品名称
**HoloGraph | 全像星图**

### 产品定位
个人知识图谱化工具 - 将碎片化知识转化为可观测、可导航、可生长的动态知识宇宙。

### 目标用户
- 知识工作者（研究员、作家、开发者）
- 终身学习者
- 需要管理复杂概念网络的专业人士

---

## 核心价值主张

> **看见，连接，成为无限。你的思想并非孤岛，而是一片等待被点亮的星群。**

HoloGraph 解决的核心问题：
1. **信息孤岛** - 笔记分散各处，难以形成整体认知
2. **线性局限** - 传统文件夹结构无法表达知识间的复杂关联
3. **检索困境** - 知道记录过但找不到，或忘记记录过什么
4. **连接盲区** - 无法发现跨领域、跨时间的隐性关联

## 功能需求

### 1. 知识节点管理 (Node Management)

#### 1.1 节点创建
- **手动创建**: 用户可通过工具栏或快捷键(Ctrl+N)创建新节点
- **属性定义**:
  - 标签 (label): 节点显示名称
  - 类型 (type): concept(概念) / note(笔记) / task(任务) / reference(引用) / tag(标签)
  - 内容 (content): 支持 Markdown 格式的详细内容
  - 位置 (position): 2D 坐标系统中的 x, y 坐标
  - 颜色 (color): 可选自定义颜色，默认根据类型自动分配

#### 1.2 节点编辑
- 双击节点打开编辑面板
- 实时预览 Markdown 内容
- 支持标签系统（逗号分隔）
- 时间戳自动记录（创建时间、最后修改）

#### 1.3 节点删除
- 支持单节点删除
- 删除时提示是否保留连接边

### 2. 知识关联管理 (Edge Management)

#### 2.1 边的创建
- **拖拽连接**: 从节点的连接点拖拽到目标节点
- **智能建议**: AI 分析节点内容，建议潜在连接

#### 2.2 边的属性
- 标签 (label): 描述连接关系的类型
- 类型 (type): related(相关) / depends(依赖) / refines(细化) / sequence(序列)
- 权重 (weight): 连接强度，影响布局算法

### 3. 可视化图谱 (Graph Visualization)

#### 3.1 画布交互
- **缩放**: 鼠标滚轮或触摸板缩放 (10% - 500%)
- **平移**: 拖拽空白区域移动视口
- **选择**: 单击选中节点，Shift+单击多选
- **框选**: 支持拖拽框选多个节点

#### 3.2 布局算法
- **力导向布局**: 模拟物理引力/斥力，自动排列节点
  - 节点间排斥力
  - 连接边产生吸引力
  - 支持实时动画
- **层次布局**: 适用于有向依赖关系
- **圆形布局**: 适用于中心辐射型知识

#### 3.3 视图控制
- **适应视图**: 一键缩放至包含所有节点
- **聚焦模式**: 选中节点时淡化其他节点
- **路径高亮**: 高亮显示两个节点间的最短路径

### 4. 智能搜索与导航 (Search & Navigation)

#### 4.1 全局搜索
- 快捷键: Ctrl+F
- 支持按节点标签、内容、标签搜索
- 实时过滤，高亮匹配节点

#### 4.2 路径规划
- 选择起点和终点
- 使用 Dijkstra 算法计算最短路径
- 可视化显示路径

### 5. 数据持久化 (Data Persistence)

#### 5.1 本地存储
- 使用 localStorage 存储图谱数据
- 自动保存（每 30 秒）
- 导出/导入 JSON 文件

#### 5.2 导入/导出
- **导出**: 完整图谱数据为 JSON
- **导入**: 从 JSON 文件恢复图谱
- **图片导出**: 将当前视图导出为 PNG

## 非功能需求

### 性能需求
- 支持 500+ 节点流畅操作
- 初始加载时间 < 3 秒
- 交互响应延迟 < 16ms (60fps)

### 可用性需求
- 直观的拖拽交互
- 清晰的视觉反馈
- 完整的键盘快捷键支持
- 新手引导流程

### 兼容性需求
- Windows 10/11
- macOS (未来支持)
- Linux (未来支持)

## 界面设计规范

### 设计语言
**Fluent Design System** - 微软流畅设计
- 深度 (Depth): 分层阴影和视差效果
- 动效 (Motion): 有意义的动画过渡
- 材质 (Material): 亚克力 (Acrylic) 效果
- 光感 (Light): 微妙的光照效果

### 色彩系统
```css
--fluent-accent: #64b5f6;           /* 主色调 - 科技蓝 */
--fluent-bg-primary: #202020;        /* 主背景 - 深灰 */
--fluent-bg-secondary: #2d2d2d;      /* 次级背景 */
--fluent-bg-tertiary: #323232;       /* 三级背景 */
--fluent-text-primary: #e3f2fd;      /* 主文本 */
--fluent-text-secondary: #b0bec5;    /* 次级文本 */
```

### 布局结构
- **左侧边栏** (280px): 文件管理、图谱列表
- **中央画布** (自适应): 图谱可视化区域
- **右侧面板** (360px): 节点属性编辑

## 技术架构

### 技术栈
- **前端框架**: React 18 + TypeScript
- **状态管理**: Zustand
- **UI 组件**: 自研 Fluent Design 组件
- **可视化**: @xyflow/react (React Flow)
- **布局算法**: D3-force
- **桌面封装**: Electron 33
- **构建工具**: Vite 5

### 项目结构
```
HoloGraph/
├── electron/          # Electron 主进程
├── src/
│   ├── components/    # React 组件
│   ├── stores/        # Zustand 状态管理
│   ├── services/      # 业务服务层
│   ├── hooks/         # 自定义 Hooks
│   ├── utils/         # 工具函数
│   └── styles/        # 样式文件
├── docs/              # 文档
└── resources/         # 静态资源
```

### 术语表
- **节点 (Node)**: 知识图谱中的概念或信息单元
- **边 (Edge)**: 节点之间的连接关系
- **图谱 (Graph)**: 节点和边构成的网络结构
- **力导向布局 (Force Layout)**: 模拟物理力的自动布局算法
- **ViewBox**: SVG 画布的视口坐标系
